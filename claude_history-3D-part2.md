## 建议和实施计划

### 关于数据模型改进

分析了 Lovable 的数据结构，给你以下建议：

#### 1. 物品类型（Category）
在 Item 中加 `category` 字段，录入时从下拉列表选择。推荐的类型列表：
```
数码产品 | 衣物 | 书籍 | 鞋类 | 出行 | 家居 | 日用品 | 医用品 | 证件 | 收藏 | 玩具 | 食品 | 其他
```
这样物品清单可以按类型筛选，IsometricView 上也能直接显示类型名称。

#### 2. 平面图数据库化
Lovable 的 floorplan.json 有以下结构：
```json
{
  "rooms": [
    { "id": "master-bedroom", "name": "主卧",
      "walls": [{x1, y1, x2, y2}, ...],
      "doors": [{x, y, width, direction}],
      "windows": [{x, y, width}]
    }
  ],
  "furniture": [
    { "id": "f1", "name": "双人床", "room": "master-bedroom",
      "x": 500, "y": 500, "width": 2000, "height": 1800 }
  ]
}
```
建议：在云数据库新建 `floorplans` 集合，存储每个 Space 的平面图数据。先导入 Lovable 的数据作为默认值。后续用户可以通过编辑器修改。

#### 3. 柜体结构（Columns + Slots 嵌套）
Lovable 的 container 结构：
```
container {
  outer_dimensions: { width, height, depth }
  columns: [
    { width: 700, slots: [
        { type: "shelf", y_start: 0, y_end: 600, items: [...] },
        { type: "drawer", y_start: 600, y_end: 1200, items: [...] }
      ]
    },
    { width: 700, slots: [...] }
  ]
}
```
这样一个 column 内可以有多个 slot（竖向分隔），多个 column 并列（横向分隔）。这就是你要的"一列中有多行"的混合布局。

建议：将我们的 flat `slots[]` 改为 `columns[{width, slots[]}]`。这是个数据模型破坏性变更，需要同步修改：
- 容器创建/编辑逻辑
- IsometricView 渲染
- 搜索逻辑
- Mock data

#### 4. 关于用户自定义平面和柜体

**平面编辑器**：用户拖拽调整房间大小和位置。这是一个独立的功能模块，建议 hackathon 之后再做。现在先用导入的固定数据。

**柜体编辑器**：用户添加/删除列（column），调整每列宽度，在列内添加分层。交互类似电子表格的"合并单元格"。同样建议后期做。

## 头像设计方案

### 方案一：柜子图标（直观）

用一个简洁的柜体轮廓，内部有分隔层和物品图标：

```
┌─────────┐
│ 📦  👕  │
├─────────┤
│ 📚  🔑  │
├─────────┤
│ 👟  💄  │
└─────────┘
```

概念：一个打开的柜子，里面分层放着不同品类的物品。传达"有序收纳"的感觉。

### 方案二：地图标记 + 收纳（核心概念融合）

一个地图定位标记（📍）的形状，里面是一个柜子或收纳盒。传达"物品定位地图"的核心概念。

### 方案三：搜索 + 空间

一个放大镜（🔍）扫过一个房间的俯视平面图。传达"快速找到物品"。

### 方案四：极简中文风

一个方形图标，用一个中文字"纳"或"寻"作为主体，配合简洁的几何线条。

### 用Emoji组合的话

最直接的选择：
- 🗄️ 文件柜 — 直观但偏办公
- 🏠 + 📦 — 家+收纳
- 📍+ 📦 — 定位+收纳

不过Emoji作为头像分辨率太低。你更倾向于哪个方向？我可以用代码生成一个SVG图标。
